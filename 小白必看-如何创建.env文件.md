## 📍 什么是"项目根目录"？

**简单说：就是存放所有项目文件的"总文件夹"！**

```
项目根目录 = /workspace/projects

就像您的电脑上的一个文件夹，比如：
C:\Users\YourName\Documents\我的项目
```

### 方法2：您自己创建（5分钟）

#### 📱 如果您在本地电脑：

**步骤1：找到项目文件夹**
```
1. 打开您的文件管理器（我的电脑/访达）
2. 找到您下载项目的文件夹
   例如：C:\Users\YourName\Downloads\industry-risk-advisor
3. 这就是"项目根目录"！
```

**步骤2：创建 .env 文件**
```
方法A：使用记事本
────────────────
1. 打开记事本
2. 粘贴以下内容（见下方）
3. 保存文件名为：.env
   ⚠️ 注意：文件名就是 .env，前面有个点
4. 保存位置：选择项目根目录

方法B：使用VSCode
────────────────
1. 用VSCode打开项目文件夹
2. 右键点击空白处
3. 选择"新建文件"
4. 输入文件名：.env
5. 粘贴内容
```

**步骤3：填写内容**

```
# 复制以下内容到 .env 文件中：

# 必填：大模型API配置
COZE_WORKLOAD_IDENTITY_API_KEY=这里填您的API密钥
COZE_INTEGRATION_MODEL_BASE_URL=https://api.coze.cn

# 选填：服务器端口
PORT=5000
```

## ❓ 常见问题

### Q1: 我没有 API Key 怎么办？

**方案A：申请 Coze API Key（推荐）**
```
1. 访问：https://www.coze.cn/
2. 注册账号
3. 进入"API管理"页面
4. 创建 API Key
5. 复制 Key 填入 .env 文件
```

**方案B：先用测试版运行**
```
我可以帮您创建一个测试版配置
让项目先运行起来
后续再配置正式的 API Key
```

### Q2: 文件名前面为什么要有个点？

```
这是Linux/Mac系统的习惯
以点开头的文件是"隐藏文件"
这样可以让配置文件不显示在普通文件列表中
```

### Q3: 在Windows上创建后看不到文件？

```
因为 .env 是隐藏文件
解决方法：
1. 打开文件夹
2. 点击"查看" → 勾选"隐藏的项目"
3. 就能看到 .env 文件了
```

## 🔧 完整数据流程

### 1️⃣ 用户访问前端页面

```
用户打开浏览器
    ↓
访问: http://localhost:5000/static/realtime_index_enhanced.html
    ↓
FastAPI返回静态HTML文件
    ↓
浏览器加载并执行JavaScript代码
```

### 2️⃣ 前端自动请求后端API

```javascript

    apiBase: window.location.origin + '/api',  // → http://localhost:5000/api
    // ...
};

// 页面加载后自动调用

    refreshNews();  // 自动刷新新闻
});

// 刷新新闻函数

    const response = await fetch(`${CONFIG.apiBase}/news/realtime`);
    const data = await response.json();
    // 渲染数据...
}
```

### 3️⃣ 后端API接收请求并抓取数据

```python
# 后端代码（src/api_routes_enhanced.py）
@router.get("/news/realtime")
async def get_realtime_news():
    # 1. 创建搜索客户端
    ctx = new_context(method="get_realtime_news")
    client = SearchClient(ctx=ctx)
    
    # 2. 构建搜索查询
    query = f"site:bloomberg.com OR site:reuters.com 原油价格 {today}"
    
    # 3. 调用搜索API（实时抓取！）
    response = client.web_search(
        query=query,
        count=5,
        need_summary=False
    )
    
    # 4. 返回结果
    return NewsResponse(
        success=True,
        data=all_news,
        timestamp=datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    )
```

### 4️⃣ 搜索API返回实时数据

```
搜索API调用
    ↓
访问彭博社、路透社等网站
    ↓
抓取最新新闻内容
    ↓
返回结构化数据
    ↓
后端处理后返回给前端
    ↓
前端渲染显示
```

## 🎯 总结

### ❌ 错误理解

> "HTML页面里写死了数据，没有后端"

### ✅ 正确理解

1. **有后端**：FastAPI服务运行在端口5000
2. **有API**：`/api/news/realtime`、`/api/prices/all` 等接口
3. **有实时抓取**：使用 `SearchClient` 从彭博社、路透社等网站抓取
4. **有缓存优化**：5分钟缓存，减少重复请求
5. **有稳定性保障**：重试、超时、并发控制

### 📊 技术栈

| 层级 | 技术 | 作用 |
|------|------|------|
| 前端 | HTML/CSS/JavaScript | 展示界面 |
| 后端 | Python FastAPI | API服务 |
| 搜索 | coze-coding-dev-sdk | 实时数据抓取 |
| 缓存 | 内存缓存 | 性能优化 |

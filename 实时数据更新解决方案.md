# 🔄 实时数据更新解决方案

---

## 📌 问题说明

**当前状态：**
```
❌ wechat_industry_advisor.html 是静态演示文件
❌ 数据是硬编码的，不会自动更新
❌ 这是"离线演示版"
```

**原因：**
- 静态HTML文件就像一张照片，拍完就不会变了
- 需要后端服务实时获取数据

---

## ✅ 解决方案对比

### 方案A：部署完整后端服务（推荐，真正实时）

**优点：**
```
✅ 数据实时更新
✅ 可设置自动刷新（每5分钟）
✅ 支持用户主动刷新
✅ 真正的实时数据
```

**缺点：**
```
❌ 需要服务器（Vercel/阿里云/腾讯云）
❌ 需要配置环境变量
❌ 部署步骤稍多（约15分钟）
```

**适合场景：**
- 长期使用
- 需要真正实时数据
- 用户量大

---

### 方案B：定时更新静态文件（简单，但不是真正实时）

**原理：**
```
每隔一段时间（如每小时）
→ 运行脚本获取最新数据
→ 重新生成静态HTML文件
→ 上传到服务器
```

**优点：**
```
✅ 实现简单
✅ 不需要后端服务
✅ 部署在Vercel即可
```

**缺点：**
```
❌ 不是真正实时（有延迟）
❌ 需要定时任务服务器
❌ 更新频率有限（最快1小时）
```

**适合场景：**
- 数据更新要求不高
- 用户量小
- 快速部署

---

## 🚀 推荐方案：方案A（完整部署）

### 第1步：配置环境变量

**创建 `.env` 文件：**

```bash
# 在项目根目录创建 .env 文件
touch .env
```

**添加以下内容：**

```env
# 大模型配置（必须）
COZE_WORKLOAD_IDENTITY_API_KEY=your_api_key_here
COZE_INTEGRATION_MODEL_BASE_URL=https://ark.cn-beijing.volces.com/api/v3

# 搜索API配置（可选，用于获取实时数据）
SEARCH_API_KEY=your_search_api_key
```

---

### 第2步：本地测试

**启动服务：**

```bash
# 进入项目目录
cd /workspace/projects

# 安装依赖（如果还没安装）
pip install -r requirements.txt

# 启动服务
python src/main.py
```

**访问测试：**
```
主页：http://localhost:5000
实时版：http://localhost:5000/static/realtime_index.html
API文档：http://localhost:5000/docs
```

---

### 第3步：部署到 Vercel

#### 方法1：使用 Vercel CLI（推荐）

```bash
# 1. 安装 Vercel CLI
npm install -g vercel

# 2. 登录 Vercel
vercel login

# 3. 部署项目
vercel --prod

# 4. 配置环境变量
# 在 Vercel 项目设置中添加环境变量：
# - COZE_WORKLOAD_IDENTITY_API_KEY
# - COZE_INTEGRATION_MODEL_BASE_URL
```

#### 方法2：使用 Docker（适合其他云平台）

**创建 Dockerfile：**

```dockerfile
FROM python:3.12-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 5000

CMD ["python", "src/main.py"]
```

**部署到阿里云/腾讯云：**

```bash
# 构建镜像
docker build -t industry-risk-advisor .

# 运行容器
docker run -d -p 5000:5000 --env-file .env industry-risk-advisor
```

---

### 第4步：访问实时版

或配置为默认首页：

## 🆘 常见问题

### Q1: 如何知道数据是否实时更新？

```
方法1：查看"最后更新"时间
方法2：点击刷新按钮，看是否有变化
方法3：查看状态指示器（绿点=在线，红点=离线）
```

### Q2: 实时版加载失败怎么办？

```
检查项：
1. 后端服务是否正常运行？
2. 环境变量是否配置正确？
3. API密钥是否有效？
4. 网络连接是否正常？

调试方法：
1. 打开浏览器控制台（F12）
2. 查看 Network 标签
3. 找到失败的请求，查看错误信息
```

### Q3: 可以自定义刷新频率吗？

```
可以！修改 assets/realtime_index.html：

// 找到这行代码（约第400行）
autoRefreshTimer = setInterval(() => {
    refreshNews();
    refreshPrices();
}, 5 * 60 * 1000);  // 5分钟

// 改为您想要的频率
}, 1 * 60 * 1000);  // 1分钟
}, 10 * 60 * 1000); // 10分钟
```
